## Развёртка проекта
- Установить `Docker Desktop` с официального источника (https://www.docker.com/products/docker-desktop). Это установит все необходимые docker-инструменты и позволит удобно управлять контейнерами на устройстве.
- Клонировать репозиторий проекта в любую удобную папку
```
git clone git@github.com:AleksandrAvdeev/short-url.git
```
- Далее в папке с проектом запустить выполнение скрипта для инициализации компонентов проекта
```
sh project-init.sh
```
- При ошибке поднятия контейнеров при занятости порта 80 может помочь команда по остановке сервера Apache, который в Linux чаще всего его занимает
```
sudo systemctl stop apache2
```
Доступ к ресурсу появится по адресу
```
http://localhost:80
```

## Задача
Реализовать небольшое REST API сервиса коротких ссылок: пользователь присылает длинный URL, получает короткий код, по короткому коду — редирект на оригинальный URL. Также есть метод со статистикой переходов.

**Методы**
1. Создание короткой ссылки
   POST /api/links

Тело запроса:

{ "url": "https://example.com/some/very/long/path" }

Ответ:

{ "code": "abc123", "short_url": "http://localhost/abc123" }

* Валидация: поле url обязательно, должно быть валидным http/https URL.
* Если такой URL уже сокращали раньше — на твой выбор: вернуть существующий код или создать новый.

2. Редирект по короткому коду
   GET /{code}

* 302-редирект на оригинальный URL.
* 404, если такого кода нет.
* Каждый редирект увеличивает счётчик переходов на 1.

3. Статистика
   GET /api/links/{code}/stats

Ответ:

```
{
    "url": "https://example.com/...",
    "code": "abc123",
    "clicks": 42,
    "created_at": "2025-06-03T12:34:56Z"
}
```

• 404, если кода нет.

**Технические требования**
* PHP 8.x.
* Laravel 10 или 11.
* Любая БД на выбор: MySQL / MariaDB / PostgreSQL / SQLite.
* Eloquent — ок (можно использовать что привычнее).
* Короткий код — 6 символов, alphanumeric, уникальный.
* README: как поднять проект, как накатить миграции и т.д.

**Желательно (бонус, не обязательно)**
* Наличие документации
* Сделать запуск из Docker, добавить Makefile
* Понятная структура коммитов в Git

**Что НЕ нужно делать**
* Фронтенд — только API.
* Авторизация, регистрация пользователей.
* Очереди, кеш, сложная архитектура.
